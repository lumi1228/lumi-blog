## 二、项目需求分析

#### 1、需求分类
* 业务型：无纸化的效率提升、
* 痛点型：梳理用户痛点、市场
* 功能型：企业客户需求，解决流量入口
* 系统需求：性能、规范、非功能性、异常处理

#### 2、工作流

* 定义：项目内容
* 分析建模：流程图、原型图、说明文档
* 验证：用户确认、技术评审、项目评审（时间、质量、资源）
* 跟踪：反馈、变更
* 形成文档

#### 3、工具

* 原型类：Axure、墨刀、MockPlus
* 流程类：Visio、PPT/KeyNote
* 功能类：各类脑图、

IDE + git的问题已经解决了，根据平时的习惯用vscode + prettier配置，会通过vscode配置git lens来查看版本库的变化。

#### 4、核心需求分析
页面与交互 -> 组件化
代码复用 -> 工程化
接口定义 -> 标准化
