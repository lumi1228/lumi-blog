## 需求分析工作流

需求分析遵循标准化流程，确保项目顺利推进：

1. **定义**：明确项目内容和范围
2. **分析建模**：通过流程图、原型图、说明文档建立模型
3. **验证**：用户确认、技术评审、项目评审（时间、质量、资源）
4. **跟踪**：持续反馈和变更管理
5. **文档化**：形成完整的需求文档

## 需求分析内容

### 核心难点

* **多维度考量**：不仅考虑功能实现，还需平衡时间、成本、质量
* **需求确认**：形成需求文档，获得用户反馈和确认
* **需求引导**：主动引导用户需求，创造价值

### 需求分析组成

需求分析包含三个层面：**业务需求**、**用户需求**、**项目需求**

![需求分析](/blog/images/l1-demand-analysis.jpg)

### 核心业务分析

业务分析遵循以下逻辑链条：

* **技术架构**：前后端分离 → 接口鉴权 → 数据通用性 → 数据库设计
* **内容管理**：内容管理 → 搜索 → 筛选
* **社区体系**：社区业务 → 积分系统 → 用户体系 → 权限控制

### 原型设计

**原型设计**是综合考虑产品目标、功能需求场景、用户体验等因素，对产品各版块、界面和元素进行合理性排序的过程。

**产品开发阶段**：想法 → 需求采集 → 功能结构 → 原型设计 → 产品需求文档

## 技术栈考量

技术选型遵循以下原则：

* **匹配度优先**：结合团队技术实力与业务需求
* **复用优先**：能复用坚决不造轮子；需要造轮子时，坚决不牵强
* **简单高效**：以简单为原则，综合考虑时间、进度、质量

![技术栈](/blog/images/l1-technology-stack.jpg)

### PC端/WebApp 技术栈

* **Vue.js**：学习曲线平缓，生态完备，约定大于配置
* **LayUI**：提供完备的社区模板
* **iView-admin**：提供成熟的管理后台模板

### 跨端技术栈

* **uni-app**：生态丰富，语法与 Vue 一致，应用成本低
* **Flutter**：性能接近原生，组件化思想成熟，技术栈完善
* **Electron**：前端开发者桌面端开发的首选方案
